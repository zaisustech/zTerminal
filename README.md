# zTerminal

A native macOS terminal built with SwiftUI + AppKit, embedding
[SwiftTerm](https://github.com/migueldeicaza/SwiftTerm) as the emulator.

> 📖 **Using the app?** See the [**User Guide**](USER_GUIDE.md) for how to use
> every feature (shortcuts, inline autosuggestions, project tasks, themes, and
> more). This README covers building and packaging.

## Features

- **Native & fast** — real VT100/xterm emulator running your login shell (zsh/bash),
  truecolor, your fonts and dotfiles (never modified).
- **Project-aware status bar** — current folder, git branch/status, runtime badges,
  last-command exit + duration, and a session timer; every item toggleable.
- **Inline autosuggestions** — dim ghost-text completion of `package.json` scripts for
  your project's detected package manager (bun/npm/pnpm/yarn); `Tab` to accept.
- **One-click project tasks** — auto-detected scripts for Node/Bun/Cargo/Go/Python/
  Maven/Gradle/.NET/Deno/Ruby/Make, plus per-project bookmarks in `.zTerminal.json`.
- **Script shortcuts & env vars** — global name→command aliases and per-tab environment
  variables, configured in Settings.
- **Themes** — Liquid Glass and translucent Blur modes, custom gradients, and a
  global→project theme cascade via `.zTerminal.json`.
- **Deep macOS integration** — Finder "Open in zTerminal", `zterminal://` URL scheme,
  Quick Look, notifications + Dock badge, and Keep Awake.

See the [**User Guide**](USER_GUIDE.md) for the full walkthrough and keyboard shortcuts.

- Top tab bar; **⌘T opens a new tab inheriting the current directory**.
- Bottom toolbar: current folder path, **Reveal in Finder** (local dirs only),
  git branch/status, runtime badges, command status, session start time, and a
  live duration timer. **Every item is individually configurable** — toggle each
  one from **Settings → Appearance → Toolbar Items**, or right-click the toolbar
  to show/hide items in place.
- Truecolor (`COLORTERM=truecolor`), Option-as-Meta, and a `zterminal://` URL
  scheme for "open at path".

There are **two build paths**:
- **Swift Package** — fast command-line builds + unit tests.
- **Xcode project** (generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen)) — the full app **plus the Finder Sync extension**, a compiled icon asset catalog, and code signing.

## Build & run

```bash
# --- SwiftPM: dev loop + tests ---
swift build
swift test
./scripts/bundle.sh && open build/zTerminal.app   # release .app (no Finder extension)

# --- Xcode: full app + Finder Sync extension + icon + signing ---
brew install xcodegen                 # once
xcodegen generate                     # writes zTerminal.xcodeproj from project.yml
xcodebuild -project zTerminal.xcodeproj -scheme zTerminal \
  -configuration Debug -derivedDataPath build/dd build
open build/dd/Build/Products/Debug/zTerminal.app
```

Requires macOS 13+ and Xcode 15+ (Swift 5.9+). The Xcode build compiles a Metal
shader from SwiftTerm, so a one-time `xcodebuild -downloadComponent MetalToolchain`
may be needed.

### Enabling the Finder "Open in zTerminal" menu

The Finder Sync extension is embedded in the Xcode-built app. After launching the
app once, enable it in **System Settings → General → Login Items & Extensions →
Finder Extensions → zTerminal**. Then right-click a folder (or a Finder window
background) and choose **Open in zTerminal**. It launches the app at that folder
via the validated `zterminal://open?path=…` URL scheme.

## Project config: `.zTerminal.json`

Drop a `.zTerminal.json` in a project folder to give it **bookmarks** — favorite or
custom shell commands, each with an icon — and an optional **per-project theme**.
When the active tab is in that directory, a **bookmark** button appears in the toolbar
(separate from the **play** button used for auto-detected script-shortcuts like npm /
cargo / make). Its popover lists the bookmarks; they run like any other task (current tab
when idle, a new tab when busy or on ⌘-activation). You can **add, edit, and delete**
bookmarks from the popover (hover a row for edit/delete); every change is written back to
`.zTerminal.json`. The bookmark button is **always** in the toolbar — opening it in a
folder without a `.zTerminal.json` lets you add the first bookmark (the file is created on
save). The play button appears only when a build manifest is detected.

```json
{
  "bookmarks": [
    { "name": "Prebuild",      "command": "expo prebuild --clean",        "icon": "hammer.fill", "color": "#4F8CFF" },
    { "name": "Clean install", "command": "rm -rf node_modules && npm ci", "icon": "trash.fill",  "color": "#FB7185" },
    { "name": "Start",         "command": "npx expo start",               "icon": "play.fill",   "color": "#10B981" }
  ],
  "theme": {
    "mode": "glass",
    "accentHex": "#EC4899",
    "gradientHexes": ["#EC4899", "#F472B6", "#DB2777", "#FB7185", "#F9A8D4"],
    "terminalScheme": "liquidGlass",
    "terminalBackgroundHex": "#160A12"
  }
}
```

- `icon` is any **SF Symbol** name (defaults to `star.fill`) and each bookmark may set
  its own `color` (hex; defaults to the app accent when omitted); `command` runs verbatim
  (`&&`, pipes, redirects allowed).
- A command may take **run-time arguments** via `<label>` placeholders. Running
  `swift test --filter <pattern>` prompts for `pattern`, then runs the substituted
  command. Repeated placeholders (`cp <src> <src>.bak`) are filled from one field; a
  command with no `<...>` runs immediately.
- The `theme` block is optional and every field within it is optional.

### Theme cascade: global → project

The active theme is resolved by layering, in increasing priority:

1. your **Settings** (⌘,) — the base;
2. a **global** `~/.zTerminal.json` `theme` — your default across every folder;
3. a **project** `<cwd>/.zTerminal.json` `theme` — wins while a tab is in that folder.

A project theme only needs to specify the fields it wants to change; everything else
falls through to the global theme, then Settings. When you `cd` into a folder with no
project theme, the **global** theme applies again; with neither file, your Settings are
in effect. Nothing here is persisted — your saved Settings are never overwritten.

## Distribution (Developer ID + notarization)

zTerminal ships **outside the Mac App Store** (a local terminal can't be sandboxed).
With an Apple Developer account and a *Developer ID Application* certificate:

```bash
# one-time: store notarytool credentials in the keychain
xcrun notarytool store-credentials zTerminal-notary \
  --apple-id "you@example.com" --team-id "TEAMID" --password "app-specific-pw"

# build → sign → notarize → staple
DEVELOPER_ID="Developer ID Application: Your Name (TEAMID)" \
TEAM_ID="TEAMID" NOTARY_PROFILE="zTerminal-notary" \
./scripts/notarize.sh
```

Hardened Runtime is on for Release (`project.yml`), with the exceptions a terminal
needs (`allow-jit`, `disable-library-validation`, …) in `Resources/zTerminal.entitlements`.

## OSC 7 shell hook (accurate current-directory tracking)

zTerminal reads the current directory from **OSC 7** escape sequences. Most shells
emit them with a small hook. Add one of the following:

**zsh** (`~/.zshrc`):

```zsh
autoload -Uz add-zsh-hook
_zterm_osc7() { printf '\033]7;file://%s%s\033\\' "$HOST" "$PWD" }
add-zsh-hook chpwd _zterm_osc7; _zterm_osc7
```

**bash** (`~/.bashrc`):

```bash
_zterm_osc7() { printf '\033]7;file://%s%s\033\\' "$HOSTNAME" "$PWD"; }
PROMPT_COMMAND="_zterm_osc7${PROMPT_COMMAND:+; $PROMPT_COMMAND}"
```

Without the hook, zTerminal falls back to the tty foreground process's working
directory (`tcgetpgrp` + `proc_pidinfo`).

## Layout

```
Package.swift
Sources/zTerminal/
  zTerminalApp.swift        # @main App, menu commands, URL scheme
  Model/SessionModel.swift  # one PTY session: CWD, start time, timer, exit state
  Model/WindowModel.swift   # tabs; new tab inherits the active tab's CWD
  Terminal/TerminalHostView.swift   # SwiftTerm bridge (spawn, env, OSC 7, exit)
  Support/CwdLogic.swift    # pure logic: OSC 7 parse, ~ abbrev, path validation
  Support/ForegroundCwd.swift       # proc_pidinfo fallback
  UI/{RootView,TabBar,BottomToolbar}.swift
Tests/zTerminalTests/       # unit tests for CwdLogic
Resources/Info.plist        # bundle metadata, zterminal:// scheme, Services decl
scripts/bundle.sh           # release build -> .app
```
